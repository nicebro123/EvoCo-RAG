# EvoCo-RAG 官方全量实验说明

本文档说明 `bash run.sh train` 默认启动的官方 full-data 实验矩阵。

当前官方实验使用 **evaluation protocol v3**。新结果写入带 `_v2` 的 study
目录，不会复用旧协议产生的低质量 checkpoint 或测试结果。v2 保证：

```text
生成 prompt 不包含 gold answer
审计 JSON 必须包含非空答案和完整必需字段
失败但证据正确的样本进入大模型纠错 SFT
测试逐样本预测写入 test_predictions*.jsonl
generator/auditor 调用和候选数按真实执行统计
```

每个 round 在大小模型更新完成后都会对完整 test split 做一次独立推理，并写出
`test_eval_round_NNN.json` 与 `test_predictions_round_NNN.jsonl`。任意一轮缺少
测试数据、模型或评估结果都会直接终止训练，不会退化为训练集 replay 指标。

官方全量实验入口：

```bash
bash run.sh train \
  --gpus 0,1 \
  --data-root ../rag_assets/rag_data/evoco_dataset_pack
```

如果机器有 8 张 GPU，推荐以 2 卡为一个实验单元，启动 4 个 worker 并行跑：

```bash
bash run.sh train \
  --gpu-pairs '0,1;2,3;4,5;6,7' \
  --data-root ../rag_assets/rag_data/evoco_dataset_pack
```

这表示每个 run 只使用一组 2 卡，例如 `0,1` 或 `2,3`。32 个 full-data
runs 会按 round-robin 分配到 4 个 GPU pair 队列里；每个队列内部顺序执行，
不同队列之间并行执行。

如果只想检查会生成哪些实验，而不真正启动训练：

```bash
bash run.sh preflight \
  --gpu-pairs '0,1;2,3;4,5;6,7' \
  --data-root ../rag_assets/rag_data/evoco_dataset_pack
```

`run.sh train` 会调用 `scripts/launch_all_experiments.sh`，默认启动 5 组
full-data studies，总共 32 个 run。所有实验会被放进一个 tmux master queue
里执行。单 2 卡模式默认 session 名称为：

```text
evoco_all_experiments
```

8 卡多 worker 模式会启动多个 session：

```text
evoco_all_experiments_g0_1
evoco_all_experiments_g2_3
evoco_all_experiments_g4_5
evoco_all_experiments_g6_7
```

查看单 2 卡训练：

```bash
tmux attach -t evoco_all_experiments
```

查看 8 卡 worker：

```bash
tmux attach -t evoco_all_experiments_g0_1
```

## 总览

```text
当前默认官方实验 =
  PopQA Llama-3-8B 主 Sweep

总计：1 组 study，4 个 full-data runs。

历史 Mistral-Nemo sweeps、多数据集迁移和机制消融仍保留为可显式启动的
legacy / optional specs，便于做横向对比，但不再是默认队列。
```

默认官方 spec 定义在：

```text
configs/experiments/popqa_llama8b_full_sweep_2gpu.yaml
```

---

## 1. PopQA Llama-3-8B 主 Sweep：4 个 Run

配置文件：

```text
configs/experiments/popqa_llama8b_full_sweep_2gpu.yaml
```

包含：

```text
main_policy_top5
cost_top3
precision_top8
audit_self_consistency
```

目的：

```text
评估 top-k、精度/成本、证据审计、分解 reward 对 PopQA 全量结果的影响。
```

## 5. PopQA 消融实验：8 个 Run

配置文件：

```text
configs/experiments/popqa_ablation_full_2gpu.yaml
```

包含：

```text
evoco_full
answer_only_reward
no_audit
no_action_policy
no_policy_heads
small_only
large_only
baseline_current_corag
```

目的：

```text
证明 EvoCo-RAG 的关键模块各自有效，包括证据审计、动作策略、policy heads、
大小模型共同训练和分解 reward。
```

这组实验最适合放在论文机制消融表里。

## 结果汇总

全部队列结束后执行：

```bash
python scripts/summarize_experiments.py \
  --root ../rag_assets/outputs/experiments
```

汇总结果写入 `../rag_assets/outputs/experiments/summary_v3/`。其中
`experiment_summary.json` 和 `experiment_summary.csv` 保留所有运行状态，
`experiment_ranking.csv` 只对完整的 protocol-v3 结果按准确率和成本排序；旧协议
和未完成运行不会进入排名。
