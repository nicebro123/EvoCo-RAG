# EvoCo-RAG 官方全量实验说明

本文档说明 `bash run.sh train` 默认启动的官方 full-data 实验矩阵。

官方全量实验入口：

```bash
bash run.sh train \
  --gpus 0,1 \
  --data-root ../rag_assets/rag_data/evoco_dataset_pack
```

如果机器有 8 张 GPU，推荐以 2 卡为一个实验单元，启动 4 个 worker 并行跑：

```bash
bash run.sh train \
  --gpu-pairs '0,1;2,3;4,5;6,7' \
  --data-root ../rag_assets/rag_data/evoco_dataset_pack
```

这表示每个 run 只使用一组 2 卡，例如 `0,1` 或 `2,3`。32 个 full-data
runs 会按 round-robin 分配到 4 个 GPU pair 队列里；每个队列内部顺序执行，
不同队列之间并行执行。

如果只想检查会生成哪些实验，而不真正启动训练：

```bash
bash run.sh preflight \
  --gpu-pairs '0,1;2,3;4,5;6,7' \
  --data-root ../rag_assets/rag_data/evoco_dataset_pack
```

`run.sh train` 会调用 `scripts/launch_all_experiments.sh`，默认启动 5 组
full-data studies，总共 32 个 run。所有实验会被放进一个 tmux master queue
里执行。单 2 卡模式默认 session 名称为：

```text
evoco_all_experiments
```

8 卡多 worker 模式会启动多个 session：

```text
evoco_all_experiments_g0_1
evoco_all_experiments_g2_3
evoco_all_experiments_g4_5
evoco_all_experiments_g6_7
```

查看单 2 卡训练：

```bash
tmux attach -t evoco_all_experiments
```

查看 8 卡 worker：

```bash
tmux attach -t evoco_all_experiments_g0_1
```

## 总览

```text
官方全量实验 =
  PopQA 主实验
+ PopQA 超参数探索
+ 多数据集迁移
+ PopQA 最终主配置对比
+ PopQA 机制消融

总计：5 组 studies，32 个 full-data runs。
```

默认官方 spec 定义在：

```text
configs/experiments/popqa_sweep_full_2gpu.yaml
configs/experiments/popqa_hparam_full_2gpu.yaml
configs/experiments/multidataset_full_2gpu.yaml
configs/experiments/popqa_full_sweep_2gpu.yaml
configs/experiments/popqa_ablation_full_2gpu.yaml
```

---

## 1. PopQA 主 Sweep：5 个 Run

配置文件：

```text
configs/experiments/popqa_sweep_full_2gpu.yaml
```

包含：

```text
cost_top3
precision_top5
precision_top8
answer_only_reward
no_audit
```

目的：

```text
评估 top-k、精度/成本、证据审计、分解 reward 对 PopQA 全量结果的影响。
```

---

## 2. PopQA 超参数探索：10 个 Run

配置文件：

```text
configs/experiments/popqa_hparam_full_2gpu.yaml
```

包含：

```text
top3_audit1_base
top5_audit3
top8_audit5
high_conf_065_top5
high_conf_085_top5
action_conf_035_top5
action_conf_055_top5
audit1_top5
audit5_top5
long_context_top5
```

目的：

```text
探索 top_k、审计候选数、证据置信度阈值、action 阈值和上下文长度。
```

---

## 3. 多数据集 Full 实验：5 个 Run

配置文件：

```text
configs/experiments/multidataset_full_2gpu.yaml
```

包含：

```text
popqa_standard
hotpotqa_distractor
nq_reader
asqa_dpr
popqa_retrieval
```

目的：

```text
验证 EvoCo-RAG 是否只适配 PopQA，还是能迁移到 HotpotQA、NQ、ASQA 等其他 RAG 数据集。
```

---

## 4. PopQA 最终 Full Sweep：4 个 Run

配置文件：

```text
configs/experiments/popqa_full_sweep_2gpu.yaml
```

包含：

```text
main_policy_top5
cost_top3
precision_top8
audit_self_consistency
```

目的：

```text
作为最终论文主结果候选，比较主配置、低成本配置、高召回配置和多审计一致性配置。
```

---

## 5. PopQA 消融实验：8 个 Run

配置文件：

```text
configs/experiments/popqa_ablation_full_2gpu.yaml
```

包含：

```text
evoco_full
answer_only_reward
no_audit
no_action_policy
no_policy_heads
small_only
large_only
baseline_current_corag
```

目的：

```text
证明 EvoCo-RAG 的关键模块各自有效，包括证据审计、动作策略、policy heads、
大小模型共同训练和分解 reward。
```

这组实验最适合放在论文机制消融表里。
